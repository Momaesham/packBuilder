Пайплайн — шаг за шагом
1
Подготовь входной ZIP-архив: cmds.json с таблицей CMD-номеров и названий моделей, папки с моделями (models_java/ и/или models_bedrock/), кастомные текстуры (textures/). При необходимости добавь ванильные текстуры блоков в textures_blocks/.
2
Выбери нужную вкладку вверху: Java 1.16–1.21.3, Java 1.21.4+ или Bedrock. Для Bedrock также укажи версию пака.
3
Загрузи ZIP — перетащи файл в область загрузки или нажми на неё.
4
Нажми «Собрать ресурс-пак». В логе будут видны все действия: какие файлы обработаны, где не нашлись пути, где были конфликты.
5
После сборки появятся кнопки скачивания. Выбери нужный вариант пака (см. ниже) и скачай конфиг для плагина, если нужно.
6
Загрузи пак на сервер или подключи локально, установи конфиг плагина и перезагрузи плагин. Готово.
Виды паков (только Java)
Серверный пак Основной

Использует предмет бумага (paper) как носитель кастомных моделей. Подключается через сервер и раздаётся всем игрокам автоматически. Именно этот пак заливается в продакшн.

Локальный (тестовый) пак Тест

Использует предмет палка (stick). Ключевая особенность: его можно подключить локально параллельно с серверным паком — они не конфликтуют, так как используют разные предметы. Это позволяет тестировать новые модели и текстуры поверх уже работающего серверного пака, не затрагивая его.

Чтобы получить предмет с нужной моделью в Java 1.21.4+:

/give @p stick[custom_model_data=НОМЕР] 1

Где НОМЕР — это CMD-номер модели из твоего cmds.json (например, 901, 902 и т.д.).

Bedrock пак Bedrock

Собирается отдельно по Bedrock-структуре. Содержит manifest.json, гео-модели и текстуры в нужных папках. Локальный вариант для Bedrock не предусмотрен.

⚠ Внимание

Тестовые модели (локальный пак) предназначены только для проверки — посмотреть как выглядит модель, проверить текстуры. Строить из них окружение или декорации на карте нельзя, кроме как для демонстрации. Итоговые модели должны проходить через серверный пак.

Структура входного ZIP-архива ▼
📦 your_pack.zip
 ├─ 📄 cmds.json
 ├─ 📁 models_java/  ← .json модели
 ├─ 📁 models_bedrock/  ← .geo.json модели
 ├─ 📁 textures/  ← текстуры (.png) кастомные
 ├─ 📁 textures_icon/  ← иконки (опционально, только Bedrock)
 └─ 📁 textures_blocks/  ← ванильные текстуры блоков (опционально), всё что есть в майне и перерисовывается текстура

Формат cmds.json — ключ: номер CMD, значение: название модели (без .json):

{
  "901": "cashier_1",
  "902": "cashier_2",
  "903": "cashier_3"
}
